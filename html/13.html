📋 Gambaran Umum

Panduan ini akan membantu Anda mengimplementasikan komponen pemindai barcode kamera yang dapat digunakan kembali yang bekerja di semua halaman POS termasuk Penjualan, Pembelian, Manajemen Stok, dan sistem POS.

Gambar: Tombol pemindai kamera terintegrasi dengan mulus dalam antarmuka POS.

Fitur-fitur

  • ✅ Kompatibilitas universal di seluruh modul
  • ✅ Deteksi halaman otomatis dan perilaku yang sesuai
  • ✅ Komponen Blade yang dapat digunakan kembali
  • ✅ Dukungan fallback untuk berbagai endpoint pencarian
  • ✅ Berbagai opsi gaya
  • ✅ Akses kamera HTTPS dengan penanganan kesalahan yang tepat
  • ✅ Antarmuka pemindaian yang dioptimalkan untuk perangkat mobile
  • ✅ Deteksi barcode secara real-time

Gambar: Antarmuka pemindaian profesional dengan umpan kamera waktu riil.

🔧 Pemecahan masalah

Kamera Tidak Berfungsi

  • Masalah: Kamera tidak mau menyala
  • Solusi: Pastikan koneksi menggunakan HTTPS
  • Periksa: Izin browser untuk akses kamera

Tombol Tidak Merespons

  • Masalah: Klik tidak membuka kamera
  • Solusi: Periksa apakah skrip dimuat dengan benar
  • Periksa: Console browser untuk kesalahan JavaScript

Kolom Input Salah

  • Masalah: Barcode masuk ke kolom input yang salah
  • Solusi: Verifikasi apakah search-input-id sesuai dengan ID input yang sebenarnya
  • Periksa: Periksa elemen untuk memastikan ID input

🚀 Fitur Lanjutan

Shortcut Keyboard

  • Tekan: Ctrl + B untuk membuka pemindai kamera
  • Berfungsi di halaman mana pun dengan tombol pemindai

Dukungan Mobile

  • Desain responsif untuk perangkat mobile
  • Tombol ramah sentuhan
  • Optimisasi kamera untuk perangkat mobile

Penanganan Kesalahan

  • Fallback yang elegan saat kamera tidak tersedia
  • Pesan kesalahan yang jelas untuk pengguna
  • Mekanisme coba ulang otomatis

📈 Tips Performa

Optimasi

  • Single Modal: Modal hanya disertakan sekali per halaman dengan @once
  • Event Delegation: Menggunakan event delegation untuk performa yang lebih baik
  • Lazy Loading: Pemindai hanya diinisialisasi saat dibutuhkan
  • Manajemen Memori: Pembersihan yang tepat saat kamera berhenti

Praktik Terbaik

  • Gunakan ID input yang spesifik untuk penargetan yang lebih baik
  • Uji di lingkungan HTTPS
  • Sediakan fallback untuk perangkat tanpa kamera
  • Jaga agar properti komponen tetap sederhana dan fokus

🔒 Pertimbangan Keamanan

Persyaratan HTTPS

  • Akses kamera memerlukan HTTPS
  • Pengembangan: Gunakan localhost atau HTTPS
  • Produksi: Pastikan sertifikat SSL tersedia

Permissions

  • Browser meminta izin akses kamera
  • Pengguna dapat menolak dan menggunakan input manual
  • Degradasi secara elegan saat izin ditolak

📱 Kompatibilitas Browser

Browser yang Didukung

  • ✅ Chrome 60+
  • ✅ Firefox 55+
  • ✅ Safari 11+
  • ✅ Edge 79+
  • ✅ Browser mobile (Safari iOS, Chrome Mobile)

Support Fallback

  • Fallback otomatis ke input manual
  • Berfungsi tanpa kamera di perangkat apa pun
  • Pendekatan peningkatan progresif
